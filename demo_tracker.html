Pupil-Only Eye Tracker

Headless, Fast, and Accurate Eye Tracking Without Face Detection

Key Features

Eye-Only Detection

Pure pupil tracking using adaptive thresholding, morphological operations, and contour detection. No face landmarks required.

Pupil-Only Calibration

Simple 5-point calibration (center, left, right, top, bottom) that takes only 10 seconds. More robust than face-based methods.

Fast Image Switching

Preloaded image cache ensures instant switching between study stimuli with zero lag or re-downloads.

ROI Tracking

Sticky search box around last detected pupil for 3-5x performance improvement and better stability.

Configurable

JSON-based configuration for threshold, smoothing, and ROI parameters. No code changes needed.

Headless Design

Pure logic module with no UI, file I/O, or dependencies. Easy to integrate anywhere.

Quick Start

# Initialize tracker from src.vision import EyeOnlyTracker, GazeMapper, CameraCapture capture = CameraCapture(source="obs") capture.open_capture() capture.calibrate_pupil_only() capture.run_loop()

Architecture

The system uses a three-layer architecture: CameraCapture handles frame acquisition and preprocessing, EyeOnlyTracker performs pupil detection with CLAHE and adaptive thresholding, and GazeMapper transforms pupil coordinates to screen space using affine transformations.

Technical Details

Algorithm

CLAHE → Gaussian Blur → Adaptive Threshold → Morphology → Contour → Moments → Moving Average → ROI

Calibration

Least-squares affine transformation fitting from 5 pupil-screen point pairs collected over 10 seconds.

Smoothing

Configurable moving average window (default 5 frames) for stable, jitter-free gaze positions.
